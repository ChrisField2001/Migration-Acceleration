# 00 - How To Use This Example Pack

This is a fictitious but realistic presales example designed to test the MCEM Uncommitted to Committed process.

## How to run the test
1. Read `01-discovery-answers.md` and validate if the input quality is enough for your current opportunity.
2. Review `06-commit-gate-check.md` to confirm stage recommendation logic.
3. Use `02-readiness-output.md` and `03-factory-nomination-pack.md` as handoff artifacts.
4. Compare this pack with your live account notes and replace the values in each section.
5. Log what worked and what did not in `04-feedback.md` and `05-engagement-log.md`.

## What to replace first for your real account
- Account name, target commit date, and buyer path
- Wave-1 scope and out-of-scope
- Top risks and owners
- Unknowns and closure dates
- Customer and Microsoft actions for the next 2 weeks

## Guardrail
Keep all content at presales qualification depth. Do not move into low-level delivery design in this stage.
