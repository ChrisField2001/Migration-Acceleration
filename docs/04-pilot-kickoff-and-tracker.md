# Pilot Kickoff and Tracker

## Purpose
Track progress of the 3 pilot accounts running through the migration acceleration framework.
This is not a delivery plan. It is a pre-sales execution tracker.

## Pilot Objectives
- Validate the 10-question discovery flow works on real accounts
- Produce at least one nomination-quality pack per account
- Capture friction and feed it into v2 improvements
- Prove the SE time cost is under 60 minutes per account

## Per-Account Execution Checklist
Copy this block for each account.

### Account: [Name]
- SE Owner:
- Customer PM contact:
- Migration objective:
- Target nomination date:

Discovery step:
- [ ] 10 core questions asked and captured
- [ ] Unknowns explicitly marked
- [ ] Conditional follow-ups triggered (list which, if any)

Scoring step:
- [ ] Each question scored 0, 1, or 2
- [ ] Weighted score calculated
- [ ] Status band confirmed: Not Ready / Needs Pre-Work / Ready to Nominate

Pre-work step:
- [ ] Top 3 blockers identified
- [ ] Each blocker has named owner and target date
- [ ] Dependency sequence defined (at least 3 steps)
- [ ] Specialist engagement triggered (see RACI doc)

Nomination pack step:
- [ ] Readiness output template filled
- [ ] Factory nomination pack template filled
- [ ] Facts separated from assumptions
- [ ] Open decisions have owner and needed-by date

Handoff step:
- [ ] Both outputs shared with CSU/Factory
- [ ] Review meeting or async review confirmed
- [ ] Rework requested? (yes/no, note reason)

Feedback step:
- [ ] 1-minute feedback form submitted
- [ ] Friction captured in backlog
